# Migration notes

The following migrations were used temporarily to support a one-time legacy import from an old database:

- `2025_10_01_000021_dedupe_legacy_objectnames_and_partof.php` — originally removed duplicates and added unique indexes. Converted to a no-op to avoid destructive behavior on fresh environments.
- `2025_10_01_000022_remove_unique_indexes_from_legacy_names.php` — originally dropped the unique indexes. Converted to a no-op after the import was completed.

Rationale:
- These migrations performed maintenance during a migration session and are not intended to run automatically on fresh installs or CI pipelines.
- The original SQL is preserved in git history; if you need to re-run them intentionally, you can retrieve the original file content from the commit where they were first added.

If you prefer, we can move these files out of the migrations folder into a maintenance scripts directory and reference them from documentation instead.
